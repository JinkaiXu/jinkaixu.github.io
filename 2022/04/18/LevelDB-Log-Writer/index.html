1. 日志写什么？

为了防止字符串的频繁拷贝，LevelDB 中字符串用 Slice 表示，其中 data_ 为字符串的初始地址，size_ 为该字符串从初始地址开始的长度。

上面介绍了表示字符串的方法，对于日志来说，我们不能直接把日志的内容写进去，因为写入的日志总归要读取对吧，所以我们需要日志的 format，在 LevelDB 中，日志逻辑上是以 block （32KB）为单位的， 假如我们要写入一个 Slice ，LevelDB 将它包装成一个或者多个（Slice 有时需要分段来分别装在不同的 block 中，这时候 Slice 被分为多个 Record） Record，它的 format 如下：

如上面所述，如果当前用于写 Log 的 block 的剩余空间装不下当前要写的 Slice 时，这时候就要将 Slice 分段，这时候一个 Slice 对应多个 Record，header 中的 RecordType 就是用来标记一个 Record 的位置情况的：

如果当前 block 只剩下 7 个bytes 的空间（刚好装下 Record 的头部）或者剩下 x(0 < x < 7) bytes 的空间时怎么处理呢？

  1. x == 7：LevelDB 会写入一个 Record，但是这个 Record 只有 header，data 部分大小为 0，Record 的 RecordType 为 kFirstType。
  2. 0 < x < 7：LevelDB 会用 '\0' 填充这个 block 剩下的空间
